# 全系统页面现代化改造设计规格

**目标：** 在不改变 API、RBAC 权限、资产字段、生命周期门禁、审批逻辑及现有模块入口的前提下，重构 IT 资产全生命周期管理系统的全局导航、页面层级、视觉语言与响应式交互。

**范围：** 覆盖仪表盘、校验看板、资产台账、采购、入库、出库、变更、故障、维保、退役、导入导出、审批中心、审批通知、运营报表、统计看板、用户、角色和配置中心；同时覆盖标签页、筛选、表格、详情、时间线、表单、弹窗、抽屉、空状态、加载状态与窄屏交互。

## 1. 设计原则

- 采用平衡型运营控制中心：统一仪表盘优先展示待办、风险和关键状态，兼顾管理员、运维工程师和管理者。
- 桌面端服务高密度台账与批量操作；窄屏端服务查询、扫码、待办、巡检、故障和审批等高频现场任务。
- 每个页面保留一个清晰的主要动作，风险、状态、责任人、时间和下一步操作必须优先可见。
- 以现有 Vue 3、Element Plus、ECharts 和图标库为基础，不引入新前端框架。

## 2. 信息架构

一级导航调整为五个稳定域，权限不足时隐藏对应模块和操作：

1. **工作台**：统一仪表盘、校验与风险。
2. **资产运营**：资产台账、采购、入库、出库、变更、故障、维保、退役。
3. **协同中心**：审批中心、审批通知、导入与导出。
4. **洞察报告**：运营报表、统计看板、阶段与风险趋势。
5. **系统治理**：用户、角色、配置中心。

现有 `currentTab` 值、路由调用、权限判断、表单字段与 API 路径保持不变。改造仅改变导航分组、显示顺序、文案层级和页面布局。

## 3. 统一页面模式

| 页面模式 | 适用模块 | 固定结构 |
| --- | --- | --- |
| 运营总览 | 工作台、校验、统计 | 待办、风险、关键指标、趋势、快捷入口 |
| 台账列表 | 资产及全部业务记录 | 页标题和主要动作、可收纳筛选、保存视图、表格、批量操作、分页 |
| 详情与处理 | 资产、变更、故障、审批 | 对象摘要、状态时间线、分组字段、关联记录、审计、操作区 |
| 配置与管理 | 配置中心、用户、角色 | 二级导航、页内标签、说明、表单、影响提示、恢复默认 |
| 现场移动端 | 后续移动/平板应用 | 搜索、扫码、待办、资产详情、巡检、报修和审批；不承载复杂配置与批量操作 |

## 4. 组件与交互规则

- **导航**：桌面端使用固定侧边栏；窄屏端收纳为抽屉。活动模块、未读审批和待办数使用统一状态表达。
- **标签页**：仅用于同一业务对象的平级内容；配置中心、审批中心等多层标签必须有清晰的域级与页内层级，不用标签页替代一级导航。
- **列表**：筛选条件默认收纳，可保存常用视图；批量操作只在选中记录后出现；表格需提供加载、空数据和错误状态。
- **详情**：顶部显示对象名称、编码、阶段、风险和主要动作；生命周期、审批链、审计记录采用时间线或顺序列表。
- **表单与弹窗**：操作类型明确、字段按业务分组、错误就近显示；复杂编辑优先使用抽屉或独立详情页，确认型操作使用简短弹窗。
- **状态语义**：正常、待处理、风险、失败、已完成、已撤回等状态保持全局一致，并且不得只依赖颜色区分。

## 5. 实施约束与验收

- 不删除或改名既有 `currentTab`、接口路径、权限判断、CRUD 入口、生命周期阶段或审批字段，除非另有明确需求与迁移方案。
- 桌面与窄屏均不得出现横向溢出、遮挡、布局抖动或无法操作的弹窗；表格窄屏采用字段优先级、详情抽屉或卡片式摘要，不简单缩小字体。
- 为公共导航、标签页、关键页面结构与响应式规则补充静态契约测试；资产 CRUD、阶段流转、审批、配置和报表执行完整回归。
- 每个实施批次完成后执行：`python -m pytest -q --disable-warnings`、`python -m compileall backend -q`、`git diff --check`，并进行浏览器级桌面/窄屏验证。

## 6. 建议实施顺序

1. 全局视觉令牌、应用框架、侧边导航、顶部操作区与标签页规范。
2. 资产台账、资产详情、采购/出入库/变更/故障/维保/退役的列表与详情模式。
3. 统一仪表盘、校验、审批中心与审批通知。
4. 报表、统计、配置中心、用户与角色。
5. 窄屏适配、全量回归、浏览器验收与可访问性修正。
