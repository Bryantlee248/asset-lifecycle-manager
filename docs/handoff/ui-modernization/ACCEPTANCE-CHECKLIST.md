# UI 现代化改造验收清单

外部 AI 必须在交付说明中逐项标记结果并提供验证证据。

## A. 功能不回归

- [ ] 所有 `PAGE-MAPPING.md` 列出的 `currentTab` 均有可访问入口。
- [ ] 既有权限条件仍隐藏无权菜单与操作按钮。
- [ ] 资产 CRUD、采购、入出库、变更、故障、维保、退役的列表、表单和详情均可用。
- [ ] 生命周期 7 阶段显示与流转行为未改变；门禁失败仍显示后端返回的原因。
- [ ] 审批创建、查看、通过、驳回、撤回、重新提交与通知未读数可用。
- [ ] 配置中心全部子标签、恢复默认、保存和缓存刷新行为未改变。
- [ ] 报表、统计与 ECharts 容器正常加载；无数据时不报错。

## B. 信息架构与视觉

- [ ] 一级导航为工作台、资产运营、协同中心、洞察报告、系统治理五个域。
- [ ] 仪表盘将待办、风险、资产状态和快捷入口置于首屏。
- [ ] 每种业务列表均具备标题、主要动作、筛选区、表格、批量操作与分页/空状态。
- [ ] 资产、故障、变更和审批详情均具备对象摘要、状态、下一步操作、关联记录与审计/时间线。
- [ ] 配置和审批中的多层标签具有明确层级；标签文本不截断。
- [ ] 正常、待处理、风险、失败、完成、撤回状态有一致的文字与色彩语义。

## C. 响应式与可访问性

- [ ] 1440px：侧栏、表格、筛选、图表、对话框无重叠或横向溢出。
- [ ] 768px：导航、统计卡片、筛选和表单正常换行，主操作仍明显。
- [ ] 375px：导航可理解地打开；资产详情、待办、审批、巡检/报修入口可操作。
- [ ] 所有图标按钮有可见文本或 `aria-label`；不以颜色作为唯一状态信息。
- [ ] 加载、空数据、请求失败和权限不足均提供可理解提示。

## D. 自动化与交付

- [ ] 已更新或新增前端静态契约测试，且没有删除既有关键断言。
- [ ] `python -m pytest -q --disable-warnings` 通过。
- [ ] `python -m compileall backend -q` 通过。
- [ ] `git diff --check` 无错误。
- [ ] 已报告修改文件、测试结果、浏览器验证、已知限制；未执行提交、推送或部署。
