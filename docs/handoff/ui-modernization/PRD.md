# PRD：IT 资产全生命周期管理系统全页面现代化改造

## 1. 背景与问题

系统已具备资产生命周期、审批、配置、报表和审计能力，但前端页面仍以单层菜单和大量同质卡片/表格为主。导航分组、标签页层级、风险和待办信息优先级不足，复杂模块在窄屏下也缺少明确的使用边界。目标是提升专业运维场景下的可读性、定位效率和后续移动端扩展性，不能牺牲现有业务功能。

## 2. 目标与成功标准

### 目标

- 将现有功能归类为稳定的五个一级业务域。
- 让所有模块使用一致的列表、详情、审批、配置和状态表达模式。
- 在统一仪表盘中同时支持管理员、运维工程师和管理者：优先展示待办、风险、关键状态和快捷入口。
- 桌面端支持高密度资产处理；窄屏端支持查询、扫码预留、待办、巡检、报修和审批等高频现场任务。

### 成功标准

- 所有既有 `currentTab` 页面都有新导航入口，且用户权限不足时仍按既有规则隐藏。
- 资产、采购、入出库、变更、故障、维保、退役、审批、配置、报表、用户和角色页面均采用已定义页面模式。
- 桌面 1440px、平板 768px、手机 375px 下无横向溢出、遮挡、不可点击控件或无法关闭的弹窗。
- 现有 API、权限检查、CRUD、生命周期流转、审批和报表回归全部通过。

## 3. 用户与核心任务

| 用户 | 高频任务 | 页面优先信息 |
| --- | --- | --- |
| 资产管理员 | 建档、盘点、入出库、批量维护、配置 | 台账、筛选、异常字段、批量动作、配置影响 |
| 运维工程师 | 查询位置、巡检、故障、变更、维保 | 当前状态、责任人、位置、风险、下一步动作 |
| 管理者/审批人 | 查看风险、处理待办、审批、观察趋势 | 待审批、风险数量、阶段瓶颈、趋势与责任归属 |

三类用户共享同一仪表盘和导航；仅按既有权限隐藏没有访问权的模块、按钮和数据。

## 4. 信息架构

1. **工作台**：统一仪表盘、校验与风险。
2. **资产运营**：资产台账、采购、入库、出库、变更、故障、维保、退役。
3. **协同中心**：审批中心、审批通知、导入与导出。
4. **洞察报告**：运营报表、统计看板。
5. **系统治理**：用户、角色、配置中心。

一级导航用于跨业务域切换；页内标签页只用于同一对象的平级内容。不得使用多层标签页替代一级导航。

## 5. 页面与组件要求

### 5.1 全局框架

- 顶部栏：系统名称、全局搜索入口、待办/通知入口、当前用户菜单。
- 桌面端：固定侧边导航，按五个一级域分组；活动态、未读数和权限隐藏行为明确。
- 窄屏：侧栏收纳为抽屉或菜单入口；不能仅缩窄成图标栏而失去文字导航。

### 5.2 统一仪表盘

- 首屏展示待审批、待处理故障、临期维保、风险资产与核心资产数量。
- 提供最近操作、阶段状态/趋势和资产运营快捷入口。
- 指标均基于现有接口数据；无数据时显示可理解的空状态，不伪造数值。

### 5.3 列表与详情

- 列表：页面标题、主要动作、可收纳筛选、保存视图入口、稳定列顺序、批量操作、分页、加载/空/错误状态。
- 详情：顶部对象摘要，明确编码、状态、位置、责任人和主要动作；使用分组字段、时间线、关联记录和审计信息。
- 复杂编辑优先抽屉或独立详情；短确认操作使用弹窗。

### 5.4 审批、配置、报表

- 审批中心默认突出“待我审批”，审批详情必须清晰显示审批链、当前节点、意见、关联资产和操作。
- 配置中心使用二级域导航和页内标签，保留恢复默认、权限限制与变更影响说明。
- 报表和统计使用统一图表容器、筛选区、指标说明和空数据状态。

## 6. 明确不允许的变更

- 不修改 API URL、请求/响应字段、数据库、后端业务逻辑、审批工作流、生命周期校验与权限命名。
- 不删除、不改名 `currentTab` 值、现有页面入口、表单字段或 CRUD 事件。
- 不写死密码、令牌、远端地址或生产数据。
- 不删除现有未提交改动、报告目录、`.env`、数据库或密钥文件。
